📘 Marco jurídico e informático

🧩 1. Visión tripartita del derecho

Todo análisis jurídico puede descomponerse en tres componentes:
```
•	Normas → reglas que indican cómo deben ser las cosas
•	Hechos → lo que efectivamente ocurre en la realidad
•	Valores → criterios para juzgar si algo es justo o correcto
```

💡 Idea clave:
El derecho surge de la interacción entre estos tres elementos.

⸻

⚖️ 2. Gobierno y regulación en informática

Incluye el conjunto de normas que regulan el uso de tecnologías:
```
•	Propiedad intelectual
•	Protección de datos personales
•	Legislación informática
```

🔹 Propiedad intelectual

Protege creaciones del intelecto (software, obras, etc.)
```
•	Derechos de autor: protegen el código como obra
•	Patentes (menos comunes en software)
•	Marcas
```

⸻

🔐 3. Protección de datos personales

Regula cómo se recolectan y usan datos de personas.

Principios básicos:
```
•	Consentimiento del usuario
•	Finalidad específica (no usar datos para cualquier cosa)
•	Seguridad en el tratamiento
```

💡 Ejemplo: una app no puede usar tus datos para fines distintos a los informados.

⸻

🌐 4. Seguridad en la red (conceptos fundamentales)

Propiedades que debe garantizar un sistema:
```
•	Integridad → los datos no deben ser alterados
•	Disponibilidad → los sistemas deben estar accesibles
•	No repudio → no se puede negar una acción realizada
```


```
•	Confidencialidad → solo acceden quienes están autorizados
```

⸻

🏛️ 5. Descentralización

Distribución del poder o control:
```
•	Competencia → qué entidad puede actuar
•	Jurisdicción → ámbito geográfico o legal donde actúa
```

💡 Muy relevante en Internet (no hay un único “dueño”).

⸻

📜 6. Derecho comercial aplicado a informática

Regula actividades económicas digitales:
```
•	Contratos electrónicos
•	Comercio online
•	Responsabilidad de plataformas
```

⸻

💻 7. Licencias de software

Definen cómo se puede usar, modificar y distribuir el software.

Tipos principales:
```
•	Copyleft
```
→ obliga a mantener la misma licencia
→ ejemplo: GPL
```
•	Permisivas
```
→ permiten reutilizar con pocas restricciones
→ ejemplos: MIT, BSD
```
•	Propietarias
```
→ código cerrado, uso restringido
→ ejemplo: software comercial

⸻

🧠 Resumen conceptual
- El derecho en informática = normas + tecnología + valores sociales
- La seguridad informática se basa en propiedades formales del sistema
- Las licencias determinan qué tan libre es el software

La regulación intenta equilibrar:
- innovación
- derechos individuales
- intereses económicos
